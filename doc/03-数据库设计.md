# 03 - 数据库设计

本章覆盖两部分：

1. **新增业务表**：指令序列表（`payload_cmd_sequence`）；
2. **菜单数据**：把业务菜单写入 `sys_menu`（遥控 / 遥测 / 单板 / LVDS / 重构）。

> **三套 SQL 同步**：仓库维护三套初始化脚本，新增表/菜单需**同时**更新：
> - `ruoyi-fastapi-backend/sql/ruoyi-fastapi-mysql.sql`（MySQL）
> - `ruoyi-fastapi-backend/sql/ruoyi-fastapi-pg.sql`（PostgreSQL）
> - `ruoyi-fastapi-backend/sql/ruoyi-fastapi-sqlite.sql`（SQLite）
>
> **同时**，当前运行库是 SQLite 文件 `ruoyi-fastapi-backend/ruoyi-fastapi.db`，需用 `sqlite3`
> 命令把同样的变更应用到现网库（见 [第 5 节](#5-落库流程)）。

## 1. sys_menu 表结构（参考）

```sql
CREATE TABLE sys_menu (
  menu_id     INTEGER PRIMARY KEY AUTOINCREMENT,
  menu_name   VARCHAR(50)  NOT NULL,   -- 菜单名称
  parent_id   INTEGER DEFAULT 0,       -- 父菜单ID
  order_num   INTEGER DEFAULT 0,       -- 显示顺序
  path        VARCHAR(200) DEFAULT '', -- 路由地址
  component   VARCHAR(255),            -- 组件路径(对应 views 下相对路径)
  query       VARCHAR(255),            -- 路由参数(JSON)，用于同组件多菜单传参
  route_name  VARCHAR(50)  DEFAULT '', -- 路由名称
  is_frame    INTEGER DEFAULT 1,       -- 是否外链(0是 1否)
  is_cache    INTEGER DEFAULT 0,       -- 是否缓存(0缓存 1不缓存)
  menu_type   CHAR(1) DEFAULT '',      -- 类型(M目录 C菜单 F按钮)
  visible     CHAR(1) DEFAULT '0',     -- 显示(0显示 1隐藏)
  status      CHAR(1) DEFAULT '0',     -- 状态(0正常 1停用)
  perms       VARCHAR(100),            -- 权限标识
  icon        VARCHAR(100) DEFAULT '#',-- 图标
  create_by, create_time, update_by, update_time, remark
);
```

> `INSERT INTO sys_menu VALUES(...)` 共 **20 列**，顺序：
> `menu_id, menu_name, parent_id, order_num, path, component, query, route_name, is_frame, is_cache, menu_type, visible, status, perms, icon, create_by, create_time, update_by, update_time, remark`。

### 角色授权

`admin`（role_id=1, role_key=`admin`）默认拥有全部权限，通常无需逐条授权。若需显式授权或为
其它角色开放，向 `sys_role_menu` 写入映射：

```sql
-- 把全部业务菜单授予角色(示例 role_id=1)
INSERT INTO sys_role_menu(role_id, menu_id)
SELECT 1, menu_id FROM sys_menu WHERE menu_id BETWEEN 2000 AND 2400;
```

## 4. 新增业务表：指令序列 `payload_cmd_sequence`

把「遥控命令 + 参数」组合成可保存、可复用的指令序列（排除广播帧）。

| 字段        | 类型          | 说明                                              |
| ----------- | ------------- | ------------------------------------------------- |
| seq_id      | INTEGER PK    | 指令序列 ID（自增）                               |
| seq_name    | VARCHAR(100)  | 序列名称                                          |
| commands    | TEXT          | 指令内容（JSON 对象数组，见下）                   |
| status      | CHAR(1)       | 状态（0 正常 1 停用）                             |
| create_by   | VARCHAR(64)   | 创建者                                            |
| create_time | DATETIME      | 创建时间                                          |
| update_by   | VARCHAR(64)   | 更新者                                            |
| update_time | DATETIME      | 更新时间                                          |
| remark      | VARCHAR(500)  | 备注                                              |

`commands` JSON 结构（对象数组）：

```json
[
  { "name": "K1502 通信激光器驱动使能", "hex": "0A 91 00 04 00 04 AA AA", "interval": 2000 },
  { "name": "K1504 工程遥测存储使能",   "hex": "0A 92 00 0A 00 0A AA AA", "interval": 2000 }
]
```

| 字段       | 说明                                                |
| ---------- | --------------------------------------------------- |
| `name`     | 指令显示名（可选，便于阅读）                         |
| `hex`      | 指令 HEX 文本（形如 `AA BB CC`）                     |
| `interval` | 发送该帧后、下一帧发送前的间隔，单位毫秒，默认 2000  |

### DDL

**SQLite**（追加到 `ruoyi-fastapi-sqlite.sql`）：

```sql
CREATE TABLE payload_cmd_sequence (
  seq_id      INTEGER PRIMARY KEY AUTOINCREMENT,
  seq_name    VARCHAR(100) NOT NULL,
  commands    TEXT         DEFAULT '[]',
  status      CHAR(1)      DEFAULT '0',
  create_by   VARCHAR(64)  DEFAULT '',
  create_time DATETIME,
  update_by   VARCHAR(64)  DEFAULT '',
  update_time DATETIME,
  remark      VARCHAR(500) DEFAULT ''
);
```

**MySQL**（追加到 `ruoyi-fastapi-mysql.sql`）：

```sql
CREATE TABLE payload_cmd_sequence (
  seq_id      BIGINT(20)   NOT NULL AUTO_INCREMENT COMMENT '指令序列ID',
  seq_name    VARCHAR(100) NOT NULL COMMENT '序列名称',
  commands    LONGTEXT     COMMENT '指令内容(JSON数组)',
  status      CHAR(1)      DEFAULT '0' COMMENT '状态(0正常 1停用)',
  create_by   VARCHAR(64)  DEFAULT '',
  create_time DATETIME,
  update_by   VARCHAR(64)  DEFAULT '',
  update_time DATETIME,
  remark      VARCHAR(500) DEFAULT '',
  PRIMARY KEY (seq_id)
) ENGINE=InnoDB AUTO_INCREMENT=1 DEFAULT CHARSET=utf8mb4 COMMENT='指令序列表';
```

**PostgreSQL**（追加到 `ruoyi-fastapi-pg.sql`）：

```sql
CREATE TABLE payload_cmd_sequence (
  seq_id      BIGSERIAL    PRIMARY KEY,
  seq_name    VARCHAR(100) NOT NULL,
  commands    TEXT         DEFAULT '[]',
  status      CHAR(1)      DEFAULT '0',
  create_by   VARCHAR(64)  DEFAULT '',
  create_time TIMESTAMP,
  update_by   VARCHAR(64)  DEFAULT '',
  update_time TIMESTAMP,
  remark      VARCHAR(500) DEFAULT ''
);
COMMENT ON TABLE payload_cmd_sequence IS '指令序列表';
```

> 对应后端 ORM 模型 `module_payload/entity/do/cmd_sequence_do.py`（`PayloadCmdSequence`，继承 `Base`）。
> 由于框架启动时会 `Base.metadata.create_all`，该表也可由 ORM 自动建表；但仍需补全三套 SQL 以保持一致。

## 5. 落库流程

```mermaid
flowchart LR
  A[编辑三套 SQL 脚本] --> B[更新现网 SQLite 库]
  B --> C[重启后端 / 重新登录]
  C --> D[前端动态菜单刷新]
```

1. **更新三套 SQL**：把菜单与建表语句分别写入 mysql/pg/sqlite 三个脚本（注意自增与时间函数差异：
   SQLite `datetime('now')`、MySQL `sysdate()`、PostgreSQL `now()`）。
2. **重建或初始化库**（全量脚本，不再维护独立 patch 文件）：

   ```bash
   cd ruoyi-fastapi-backend
   sqlite3 ruoyi-fastapi.db < sql/ruoyi-fastapi-sqlite.sql
   ```

   MySQL / PostgreSQL 分别执行 `sql/ruoyi-fastapi-mysql.sql`、`sql/ruoyi-fastapi-pg.sql`。
3. **验证**：

   ```sql
   SELECT menu_id, menu_name, parent_id, menu_type FROM sys_menu WHERE menu_id >= 2000 ORDER BY menu_id;
   SELECT name FROM sqlite_master WHERE type='table' AND name='payload_cmd_sequence';
   ```
4. **重新登录**：前端通过 `/getRouters` 拉取动态菜单，新菜单即出现在侧边栏。

> 地检平台相关 DDL（`payload_cmd_sequence`、遥测归档表、菜单 2000–2400、定时任务 100）已合入上述三套基础 SQL，重建库即可。
