# 06 - 配置文件说明

后端需读取两个配置文件（当前位于 `ruoyi-fastapi-backend/assets/config/`，路径可调整）：

| 文件                 | 用途                                       | 规模           |
| -------------------- | ------------------------------------------ | -------------- |
| `TeleControlCfg.json`| 遥控配置，构建指令树、组装 CAN 指令          | ~5.7K 行       |
| `TeleMetryCfg.json`  | 遥测配置，解析遥测数据、生成遥测菜单/表单     | ~14.3K 行      |

> 遥测类型：CAN 遥测 `FF/FD/FB/F9/F7/FE/FC`；工程遥测 `7E9B/7E9D/7E9F`。

---

## 1. TeleControlCfg.json（遥控配置）

### 1.1 顶层结构

```json
{
  "datetime": "2025-09-03 16:19:34",
  "order": { "K1501": { …指令定义… }, "K1502": { … } },
  "page":  [ { "id": …, "name": "分类名", "orderList": ["K1501", …] }, … ]
}
```

| 字段       | 类型   | 说明                                                       |
| ---------- | ------ | ---------------------------------------------------------- |
| `datetime` | string | 配置生成时间                                               |
| `order`    | object | 指令字典，键为指令代号（如 `K1501`），值为指令定义         |
| `page`     | array  | 分类/页列表，每页含若干指令代号（构建左侧指令树的分类层）  |

> `page` 用于左栏树的「分类」层（如「激光B-电源控制」），`orderList` 为该分类下的指令代号集合。
> （C++ 端以 `page → orderList → order` 三级组织。）

### 1.2 指令定义（order 项）

```json
"K1502": {
  "id": "K1502",
  "name": "通信激光器驱动使能",
  "binfile": 0,
  "timer": 0,
  "check": "否",
  "component": [ { …组件… } ]
}
```

| 字段        | 类型   | 说明                                                                 |
| ----------- | ------ | ------------------------------------------------------------------- |
| `id`        | string | 指令代号                                                            |
| `name`      | string | 指令名称                                                            |
| `binfile`   | int    | 非 0 表示支持「从文件加载/保存参数」（显示 Load/Save）              |
| `timer`     | int    | >0 表示支持定时发送，默认间隔=该值(ms)；0 不显示定时 UI            |
| `check`     | string | 「是/否」。**C++ 端未使用此字段**，校验和由组帧规则自动处理（见 07）|
| `component` | array  | 参数组件数组，按顺序拼接为指令数据                                  |

### 1.3 组件定义（component 项）

```json
{
  "title": "捕获模式",
  "componentType": "select",
  "dataType": "UINT16",
  "unit": "",
  "minVal": "",
  "maxVal": "",
  "defaultVal": "",
  "options": { "0xAAAA": "捕跟模式", "0x6666": "自标校模式" }
}
```

| 字段            | 说明                                                                    |
| --------------- | ----------------------------------------------------------------------- |
| `title`         | 组件标题（作为输入框 label / 提示）                                      |
| `componentType` | 组件类型：`fixed` / `number` / `select` / `scientific`                   |
| `dataType`      | 数据类型：`INT8/16/24/32`、`UINT8/16/24/32`、`FLOAT`、`DOUBLE` 等         |
| `unit`          | 单位                                                                     |
| `minVal/maxVal` | 数值范围（number 用，空表示不限）                                        |
| `defaultVal`    | 默认值（fixed 为 HEX 串；number/select 为默认值/默认项）                 |
| `options`       | 选项字典（select 用，键为 HEX 值，值为显示文本）                         |

### 1.4 组件类型分布与示例

> 统计自现网配置：`fixed`×206、`number`×153、`select`×35、`scientific`×24。

| 类型         | 含义               | 关键字段                          | 组帧编码（详见 07）                |
| ------------ | ------------------ | --------------------------------- | --------------------------------- |
| `fixed`      | 固定值（不可编辑） | `defaultVal`=完整 HEX             | 直接取 `defaultVal` 字节          |
| `number`     | 数值输入           | `dataType`、`minVal/maxVal`       | 按 dataType 大端编码              |
| `select`     | 下拉选择           | `options`（HEX→文本）             | 取选中项的 HEX 键为字节           |
| `scientific` | 科学计数法         | `dataType`=FLOAT/DOUBLE           | IEEE754 大端编码                  |

**fixed 示例**：

```json
{ "title": "固定值", "componentType": "fixed", "defaultVal": "0A9100040004AAAA" }
```

**number 示例**：

```json
{ "title": "X轴位置", "componentType": "number", "dataType": "INT16",
  "unit": "", "minVal": "", "maxVal": "", "defaultVal": "" }
```

**select 示例**：见 1.3。**scientific 示例**：

```json
{ "title": "注入轨道参数参考时间", "componentType": "scientific", "dataType": "DOUBLE" }
```

## 2. TeleMetryCfg.json（遥测配置）

### 2.1 顶层结构

```json
{
  "datetime": "2026-06-17 09:17:32",
  "table": { "FF": { …遥测表… }, "FD": { … } },
  "page":  [ { "id": 255, "key": "FF", "name": "0xFF：B-1主要包" }, … ]
}
```

| 字段       | 类型   | 说明                                              |
| ---------- | ------ | ------------------------------------------------- |
| `datetime` | string | 配置生成时间                                       |
| `table`    | object | 遥测表字典，键为数据类型 HEX（`FF`…），值为表定义  |
| `page`     | array  | 遥测页列表，用于**生成遥测二级菜单**               |

### 2.2 page（遥测菜单来源）

```json
"page": [
  { "id": 255, "key": "FF", "name": "0xFF：B-1主要包" },
  { "id": 253, "key": "FD", "name": "0xFD：B-2捕跟同轴标校包" },
  { "id": 251, "key": "FB", "name": "0xFB：B-3算轨包" },
  { "id": 249, "key": "F9", "name": "0xF9：B-4-1指向标校包" },
  { "id": 247, "key": "F7", "name": "0xF7：B-4-2星敏遥测包" },
  { "id": 254, "key": "FE", "name": "0xFE：算轨异步包1" },
  { "id": 252, "key": "FC", "name": "0xFC：算轨异步包2" }
]
```

| 字段   | 说明                                  |
| ------ | ------------------------------------- |
| `id`   | 数据类型的十进制（如 255 = 0xFF）     |
| `key`  | 数据类型 HEX 串（用于取表与解析）     |
| `name` | 菜单/页显示名                         |

> 这 7 项即 [03 章](./03-数据库设计.md) 的遥测二级菜单（menu_id 2101-2107）来源。

### 2.3 遥测表（table 项）

```json
"FF": { "id": "FF", "name": "B-1主要包", "row": [ { …字段… } ] }
```

| 字段   | 说明                       |
| ------ | -------------------------- |
| `id`   | 数据类型 HEX               |
| `name` | 表名                       |
| `row`  | 遥测字段数组               |

### 2.4 遥测字段（row 项）

```json
{
  "no": 8, "id": "JGB008", "name": "广播时刻（秒）",
  "bytepos": 13, "bits": 32, "bitpos": 0,
  "showType": 0, "formula": "D+8*3600", "unit": "",
  "fmt": "%time", "value": {}, "dataType": "UINT32",
  "variableName": "unBroadcastTimeUtcSec"
}
```

| 字段           | 说明                                                                        |
| -------------- | --------------------------------------------------------------------------- |
| `no`           | 序号（C++ 表中不展示，可用于排序）                                           |
| `id`           | 字段 ID（如 `JGB008`）                                                       |
| `name`         | 字段名称                                                                     |
| `bytepos`      | 字节偏移（在 payload 中的起始字节）                                          |
| `bits`         | 位宽（8/16/24/32/64；<8 表示位域）                                           |
| `bitpos`       | 位偏移（位域时使用）                                                         |
| `showType`     | >0 时显示「公式结果(原始结果)」                                              |
| `formula`      | 计算公式，`D` 代表原始值；用 tinyexpr 类表达式求值（见 07）                  |
| `unit`         | 单位（拼接在显示值后）                                                       |
| `fmt`          | 格式化串（`%d`/`%u`/`%hu`/`%04X`/`%time` 等，C 风格）                        |
| `value`        | 取值映射字典（位串或 `0x..` 键 → 文本，如 `终端模式→待机模式`）              |
| `dataType`     | 数据类型（`BYTE/UINT16/UINT32/INT16/DOUBLE…`）                              |
| `variableName` | C++ 变量名（参考，移植无需使用）                                             |

### 2.5 显示值优先级（解析时）

1. 若 `value` 非空：按位串/HEX 键查表 → 显示映射文本；
2. 否则若 `formula != "D"`：用 tinyexpr 求值，按 `fmt` 格式化；
3. 否则：原始值按 `fmt` 格式化；
4. `fmt == "%time"`：按「值 + 纪元(2020-01-01)」转 UTC 时间字符串；
5. `showType > 0`：显示 `公式结果(原始结果)`；
6. 末尾拼接 `unit`。

> 解析由 `telemetryparser` 库完成（见 [02](./02-数据采集层设计.md)、[07](./07-遥控帧组装与遥测解析规则.md)），
> 前端只展示库返回的 `id/name/show/hex/unit`。

## 3. 配置文件位置与读取

- 当前位置：`ruoyi-fastapi-backend/assets/config/{TeleControlCfg,TeleMetryCfg}.json`。
- 读取方：
  - **主进程**：读取 `page/table/order` 元信息，提供给前端构建树/表单、生成菜单；
  - **采集进程**：`telemetryparser.TeleMetryCfgManager.init(path)` 加载后解析遥测帧。
- 路径建议集中到后端配置项（如环境变量或 `config`），避免硬编码；两个进程读同一路径。
- 配置可能更新（`datetime` 标识版本），建议提供「重新加载配置」能力（**待确认**）。
