# 08 - 开发计划与里程碑

按「先文档、后编码」「可分步骤输出」的原则，将实现拆分为可独立验收的阶段。
每阶段产出可运行、可演示的增量。

---

## 1. 阶段总览

```mermaid
flowchart LR
  P0[P0 地基] --> P1[P1 遥控]
  P1 --> P2[P2 遥测]
  P2 --> P3[P3 曲线]
  P3 --> P4[P4 单板/相机]
  P4 --> P5[P5 LVDS工程遥测]
  P5 --> P6[P6 重构/收尾]
```

| 阶段 | 主题             | 核心交付                                                         |
| ---- | ---------------- | ---------------------------------------------------------------- |
| P0   | 地基             | 依赖安装、菜单/表落库、`module_payload` 骨架、Redis 接入、进程管理器 |
| P1   | 遥控             | 控制开关、遥控（树+单条指令+历史）、指令序列、CAN 采集进程         |
| P2   | 遥测             | 7 个遥测表页、遥测查询接口、遥测解析入 Redis                      |
| P3   | 遥测曲线         | 全字段时间序列落库 + ECharts 曲线 + 缩放 + 表页跳转               |
| P4   | 单板/相机        | 串口采集进程、相机图像协议、相机测试页                            |
| P5   | LVDS 工程遥测    | 高速信号解析、波形页（8 路 + 游标测量）                           |
| P6   | 重构/收尾        | 重构空白页、健壮性、文档校正、联调与验收                          |

---

## 2. 阶段任务拆解

### P0 — 地基

- [ ] 后端依赖补充：`pyserial` + 本地 wheel `gpcan`、`telemetryparser`（`whl/`）。
- [ ] 整理 `sql/patch_payload.sql`，更新三套 SQL；用 `sqlite3` 应用到 `ruoyi-fastapi.db`（见 [03](./03-数据库设计.md)）。
- [ ] 新建 `module_payload` 骨架（controller/service/dao/entity/collectors/cfg/redis_keys）。
- [ ] Redis 连接复用框架 `app.state.redis`；定义 `redis_keys.py` 命名规范（见 [02](./02-数据采集层设计.md)）。
- [ ] 采集进程管理器：启停、心跳、回收；CAN「按卡聚合 + 通道引用计数」。
- [ ] 配置加载：`TeleControlCfg/TeleMetryCfg` 路径集中化，主进程与采集进程共用。

**验收**：后端启动正常；登录后侧边栏出现「遥控/遥测/单板/LVDS/重构」菜单；各页为占位页可打开。

### P1 — 遥控

- [ ] `device_controller`：CAN 通道 open/close/list/status；串口 open/close/list。
- [ ] CAN 采集进程：初始化、收发、就地解析、指令队列、历史、状态/心跳。
- [ ] `telecontrol_controller`：config 读取、单条 send、history、control/op。
- [ ] 组帧实现：fixed/number/select/scientific 编码 + 校验和 + 广播识别（见 [07](./07-遥控帧组装与遥测解析规则.md)）。
- [ ] 前端：控制开关页（同行布局）、遥控页（三栏：树/单条指令/历史）、指令序列 CRUD + 复制。
- [ ] `sequence_controller` + 表 `payload_cmd_sequence`。

**验收**：可打开 CAN 通道；选指令、填参数、HEX 实时预览、发送成功并在历史显示；序列可增删改查与复制。

### P2 — 遥测

- [ ] CAN 采集进程接入 `telemetryparser`，按类型解析写 `payload:{id}:tm:{type}`。
- [ ] `telemetry_controller`：config、table（最新值）、fields。
- [ ] 前端遥测表页（共用组件 + `query.type`）：列表、轮询刷新、变化高亮、映射值显示。

**验收**：7 个遥测菜单分别展示对应表的实时数据；值变化高亮；映射值（如终端模式）正确。

### P3 — 遥测曲线

- [ ] 曲线落库：帧到达后全字段写入 ZSet（限量 `CURVE_MAX_POINTS`），与前端显示无关。
- [ ] `curve/data` 拉取；前端 ECharts 折线 + `dataZoom` 缩放。
- [ ] 遥测表页「点数值跳转曲线」联动（预选 + 自动确认）。

**验收**：选表/选量/确认后实时出曲线；可框选缩放；从表页点数值可直接进入并绘制。

### P4 — 单板 / 相机

- [ ] 串口采集进程：`EB 90` 协议、整图重组、重试、写 `payload:serial:{port}:image`。
- [ ] `camera_controller`：start/stop/image/status。
- [ ] 前端相机测试页：串口/分辨率/图像号选择、开始/停止、图像展示、状态栏。

**验收**：选串口与分辨率后开始刷新，能稳定显示灰度图像并按周期刷新；异常有提示与重试。

### P5 — LVDS 工程遥测

- [ ] 确认工程遥测协议与信号清单（`7E9B/7E9D/7E9F`）。
- [ ] 采集进程解析高速信号写时间序列；限频/限量与性能优化。
- [ ] 前端波形页：8 路信号选择、ECharts 高性能绘制、游标测量（ΔT/ΔY/Freq）、采样率/FPS 显示。

**验收**：可选最多 8 路信号实时波形；游标测量数值正确；高数据量下渲染流畅。

### P6 — 重构 / 收尾

- [ ] 重构空白占位页。
- [ ] 健壮性：断线重连、进程崩溃自愈、错误提示统一。
- [ ] 文档校正（把实现中确认的 **【待确认】** 项回填）。
- [ ] 端到端联调与验收。

## 4. 风险与对策

| 风险                         | 对策                                                            |
| ---------------------------- | --------------------------------------------------------------- |
| 无真实硬件联调               | 用 `CanVendorType.DEMO`（vendor=0）虚拟设备与样例帧自测          |
| 高速波形性能瓶颈             | Redis 限量 + 前端抽稀/Canvas 直绘；先小数据量打通再优化          |
| 多进程在 Windows 的启动开销  | 采集进程用独立脚本 + `spawn`；避免频繁启停，按设备生命周期管理   |
| 三套 SQL 与现网库不一致      | 统一补丁文件 `patch_payload.sql`，落库后用查询校验              |
| 协议细节缺失                 | 以 `test/` 参考实现为准对拍；不确定处标 **【待确认】** 并先桩实现 |

---

## 5. 编码阶段建议顺序

1. 先 **P0 地基** + **P1 遥控**（打通「前端 → API → Redis → 采集进程 → 设备」全链路）；
2. 再 **P2 遥测**（复用 P1 的 CAN 进程与 Redis 通道）；
3. 其余阶段按依赖推进。

> 每完成一个阶段，回到本目录更新对应文档与 **【待确认】** 项，保持「文档与实现同步」。
