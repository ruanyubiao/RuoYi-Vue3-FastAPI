# 12 · 数据解析绑定与来源归档重构

> 决策（2026-07-20）：**继续 MySQL RANGE 分区**；**不建类型字典表**（`data_kind` / `data_sub` / `src_kind` 等用代码常量 + 业务表字符串）。

| # | 目标 | 说明 |
| - | ---- | ---- |

## 二、已拍板决策

| 项 | 结论 |
| -- | ---- |
| 类型字典表 | **不建**；常量写在代码，业务表存字符串 |
| 冷数据切分 | **继续分区**（`PARTITION BY RANGE (ts_ms)`） |
| 主键 | MySQL 分区要求：`PRIMARY KEY (id, ts_ms)`（单列自增无法分区） |
| Redis | 仍作热层：最新帧 + 近期曲线 |

## 四、运行时：设备会话与绑定

### 4.1 会话模型（Redis，进程内可镜像）

```text
payload:session:{src_kind}:{src_param}
  {
    "srcKind": "can",
    "srcParam": "can:0:0:0",
    "parserId": "tm_can_yc",   # 空 = 未绑定
    "openedAt": "...",
    "status": "running"
  }
```

规则：

1. 打开设备 → 写会话，`parserId` 默认可空或按通道类型给默认值。
2. 绑定/换绑 API：设置 `parserId`。
3. 采集回调：查会话；**无 `parserId` 则不解析、不写 Redis 业务键、不入归档队列**。
4. 关闭设备 → 清会话。

### 4.2 与遥测页关系

- 页面不再要求「必须选中某个已连接 CAN」。
- 匹配条件：`data_kind=tm` 且 `data_sub=当前页类型`（如 FF）。
- 头部 Tag：显示最近一帧的 `src_param`（如 `can:0:0:0` / `http:devtest`）；无数据时显示「无数据」。

## 五、热层 Redis 键（演进）

| 键 | 含义 |
| -- | ---- |
| `payload:tm:{data_sub}:latest` | 该子类型**最新一帧**（跨来源，后写覆盖） |
| `payload:tm:{data_sub}:curve:{field}` | 近期曲线 ZSet |
| 帧 JSON 内 | 必含 `srcKind` / `srcParam` / `parserId` / `dataKind` / `dataSub` |

不再写入 `payload:{deviceId}:tm:{TYPE}` 等旧设备键。

## 六、冷层表结构（分区，演进自 doc/11）

### 6.1 `payload_tm_frame`

```sql
CREATE TABLE payload_tm_frame (
  id           BIGINT       NOT NULL AUTO_INCREMENT,
  ts_ms        BIGINT       NOT NULL COMMENT '帧时间戳(ms)',
  data_kind    VARCHAR(16)  NOT NULL COMMENT '数据大类：tm/...',
  data_sub     VARCHAR(16)  NOT NULL COMMENT '子类型：FF/FC/...',
  src_kind     VARCHAR(16)  NOT NULL COMMENT '来源：can/serial/udp/http',
  src_param    VARCHAR(128) NOT NULL COMMENT '来源参数',
  parser_id    VARCHAR(64)  DEFAULT NULL COMMENT '解释器ID',
  raw_hex      MEDIUMTEXT   NOT NULL,
  parsed_json  JSON         NOT NULL,
  field_count  INT          NOT NULL,
  cfg_version  VARCHAR(64)  DEFAULT NULL,
  created_at   DATETIME(3)  DEFAULT CURRENT_TIMESTAMP(3),
  PRIMARY KEY (id, ts_ms),
  KEY idx_kind_sub_ts (data_kind, data_sub, ts_ms),
  KEY idx_src_ts (src_kind, src_param, ts_ms)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
PARTITION BY RANGE (ts_ms) (...);
```

### 6.2 `payload_tm_field_num`

主键：`(src_param, data_sub, field_id, ts_ms)`（含分区列 `ts_ms`）。

### 6.3 `payload_tx_log`（遥控发送）

```sql
CREATE TABLE payload_tx_log (
  id           BIGINT       NOT NULL AUTO_INCREMENT,
  ts_ms        BIGINT       NOT NULL,
  src_kind     VARCHAR(16)  NOT NULL COMMENT '发送通道类型',
  src_param    VARCHAR(128) NOT NULL COMMENT '发送通道参数',
  cmd_name     VARCHAR(128) DEFAULT NULL,
  order_id     VARCHAR(64)  DEFAULT NULL,
  raw_hex      TEXT         NOT NULL,
  success      TINYINT      NOT NULL DEFAULT 1,
  message      VARCHAR(500) DEFAULT NULL,
  operator     VARCHAR(64)  DEFAULT NULL COMMENT '登录用户，可选',
  created_at   DATETIME(3)  DEFAULT CURRENT_TIMESTAMP(3),
  PRIMARY KEY (id, ts_ms),
  KEY idx_src_ts (src_kind, src_param, ts_ms),
  KEY idx_ts (ts_ms)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
PARTITION BY RANGE (ts_ms) (...);
```

  2. **UDP 监听**：选择**已打开**的 UDP 设备，绑定/解绑 `parserId`（与控制页会话同一套 Redis）。

## 七附、解释器模型澄清（2026-07-21）

> 本节沉淀讨论结论，供后续实现 **CAN 遥控响应解析** 时查阅。  
> **【后续】CAN 遥控响应（data_kind=tc）** — 见文末标记。

### 概念对照

| 概念 | 字段 | 说明 |
|------|------|------|
| 大类型 | `data_kind` | 如 `tm` 遥测、`tc` 遥控回执（预留）、`eng` / `image` |
| 子类型 | `data_sub` | 如遥测 `FF`/`FD`…；**由解释器从帧内解析**，不是绑定时手选 |
| 解释器实现 | `parserId` | 注册表键，如 `tm_can_yc`；**会话绑定的是实现，不是抽象大类型** |

关系：

```text
设备会话绑定 → parserId（具体实现，如 tm_can_yc）
                      ↓ 解析每帧
                data_kind（实现约定/帧分流）
                data_sub  （帧内字段，如第 3 字节 → FF）
```

当前 `tm_can_yc`：固定产出 `data_kind=tm`，`data_sub` 随帧变化（多种子类型）。

### 原始数据 vs 解析数据

```text
任意通道收包
    → 必写原始 IO：payload:{deviceId}:io     （与是否绑定解释器无关）
    → 若会话有 parserId
         → 解释器校验/解析
         → 写遥测热层 payload:tm:{dataSub}:* 与归档队列
    → 若无 parserId：不解析、不写业务键、不归档
```

硬件（can / serial / udp）只负责 raw；HTTP 注入是另一路 `src_kind=http` 的模拟来源。

### 同一解释器能否产出多种数据类型？

| 层级 | 是否支持 | 现状 |
|------|----------|------|
| 多种 `data_sub` | ✅ | `tm_can_yc` 已按帧产出 FF/FD/FB… |
| 多种 `data_kind` | ✅ 设计允许 | 常量已预留 `tc`；**当前实现未做** |

同一 CAN 口既有**遥测复合帧**又有**遥控请求响应帧**时：

- **推荐**：一个总线向解释器（或扩展现有实现）按帧头/类型分流 → `tm` / `tc`。  
- 会话仍是「一设备一个 `parserId`」；不要做成只绑 `data_kind`。  
- 备选：两个解释器（需改多绑定模型，当前不做）。

## 八、分区维护说明（`PARTITION BY RANGE`）

**不会在插入数据时自动新建月份分区。**

| 步骤 | 谁做 | 说明 |
| ---- | ---- | ---- |
| 建表时 | SQL 脚本 | 预置 `p202607` / `p202608` / `pmax` |
| 每月扩分区 | 定时任务 | `module_task.payload_tm_partition_job.job`（`sys_job` id=100，每月 1 日 02:00）对 `payload_tm_frame` / `payload_tm_field_num` / `payload_tx_log` 执行 `REORGANIZE PARTITION pmax` |
| 兜底 | `pmax` | 未拆出的月份数据先进 `pmax`，任务跑完后再拆到命名分区 |

因此：**DDL 里的 PARTITION 是声明式的；跨月分区靠定时任务维护，不是 MySQL 自动按月建表。**  
SQLite / PostgreSQL 无此分区，任务会跳过。
